All colors resolved from `var(--*)` tokens declared in `packages/editor/src/theme/tokens.ts`. Values below are dark theme references; light theme equivalents are in `LIGHT_THEME_TOKENS`.

| Role | CSS Token | Dark Value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `var(--background)` | `#090912` | VizPanel canvas background, overall container background |
| Secondary (30%) | `var(--surface)` | `#0f0f1a` | VizPicker strip background, toolbar background, view zone container |
| Elevated surface | `var(--surface-elevated)` | `#14141f` | VizPicker active button fill |
| Accent (10%) | `var(--accent)` | `#8b5cf6` | VizPicker active mode indicator border/outline only |
| Border | `var(--border)` | `rgba(255,255,255,0.08)` | VizPicker strip bottom border, VizPanel top border |
| Foreground | `var(--foreground)` | `#e2e8f0` | VizPicker icon color (idle and active) |
| Foreground muted | `var(--foreground-muted)` | `rgba(255,255,255,0.4)` | Inactive VizPicker icon color |
| Destructive | `#f87171` (hardcoded, matches Toolbar.tsx) | — | Error badge text only |

### Pianoroll Note Colors (Canvas 2D — not CSS)

Note block fill colors are drawn directly on canvas using resolved CSS token values read at paint time:

| Category | CSS Token | Dark Hex | Trigger condition |
|----------|-----------|----------|-------------------|
| Drums | `var(--stem-drums)` → `--warning` mapping | `#f97316` | `s` field matches drum sound list |
| Bass | `var(--stem-bass)` → `--info` mapping | `#06b6d4` | `s` field matches bass sound list |
| Melody | `var(--stem-melody)` / `var(--accent)` | `#a78bfa` | `s` field matches melody/synth sounds |
| Pad | `var(--stem-pad)` → `--success` mapping | `#10b981` | `s` field matches pad sounds |
| Default / unknown | `var(--accent)` | `#8b5cf6` | No `s` category match |
| User-defined | `hap.value.color` (literal string) | varies | `hap.value.color` present — takes priority over all above |

**Source:** `02-CONTEXT.md` decisions, `tokens.ts` stem color tokens.

### Accent Reserved For

Accent (`var(--accent)`) is reserved for exactly these elements in Phase 2:
1. VizPicker active-mode button: `outline: 1px solid var(--accent)` + `background: var(--accent-dim)` fill
2. Default/unknown-category note blocks on the pianoroll canvas
3. Melody-category note blocks on the pianoroll canvas (via `var(--stem-melody)` which is `#a78bfa`, a lighter accent shade)

Accent is NOT used for: hover states, border fills, labels, or icons.

## Component Inventory

### VizPicker (new — Phase 2)

- Container: `height: 32px`, `display: flex`, `alignItems: center`, `gap: 4px`, `padding: 0 8px`, `background: var(--surface)`, `borderBottom: 1px solid var(--border)`
- Each button: `width: 32px`, `height: 24px`, `borderRadius: 4px`, `border: none`, `cursor: pointer`, `display: flex`, `alignItems: center`, `justifyContent: center`
- Idle state: `background: transparent`, icon color `var(--foreground-muted)`
- Active state: `background: var(--accent-dim)`, `outline: 1px solid var(--accent)`, icon color `var(--foreground)`
- Icons: inline SVG, `width: 14px`, `height: 14px`, `fill: none`, `stroke: currentColor`, `strokeWidth: 1.5`
- Five modes: pianoroll, scope, spectrum, spiral, pitchwheel (scope/spectrum/spiral/pitchwheel are scaffolded, click-selectable, canvas stays blank)
- Default active mode on mount: pianoroll

### VizPanel (new — Phase 2)

- Container: `height: vizHeight` (default 200px), `background: var(--background)`, `borderTop: 1px solid var(--border)`, `overflow: hidden`, `position: relative`
- p5.js canvas fills container 100% width × 100% height via `ResizeObserver` + `p.resizeCanvas(w, h)`
- When audio is stopped: canvas remains mounted, draws nothing (blank/idle — no placeholder text, no unmount)
- Canvas background: `#090912` (dark) — draw call uses `var(--background)` resolved value

### Pianoroll Sketch (new — Phase 2, inside VizPanel)

- Time window: 6 seconds rolling (right edge = now, scrolls left) — hardcoded, not configurable
- Y-axis: MIDI 24 (C1) at bottom → MIDI 96 (C7) at top (pitch area occupies top 80% of canvas height)
- Drum lane: bottom 20% of canvas height, fixed positions per percussion sound name
- No Y-axis labels, no piano key overlay — clean canvas
- Note block: filled rectangle, `opacity: 0.85`, no stroke, `borderRadius` not applicable (canvas rect)
- Note block height: `(pitch_area_height / 72)` px per MIDI step — one pixel row minimum
- Note block width: proportional to `audioDuration` mapped to time window pixels

### Inline View Zone (new — Phase 2, inside Monaco)

- Height: 120px fixed
- Width: fills Monaco content area (Monaco view zone API handles this)
- Background: `var(--background)` resolved value (`#090912` dark)
- p5 canvas embedded — same pianoroll sketch data, same note coloring
- Re-added after every `evaluate()` call (Monaco view zones reset on re-layout)
- Displays all haps from all tracks (no per-line filtering in v1)

## Interaction States

| State | Element | Visual treatment |
|-------|---------|-----------------|
| Idle (stopped) | VizPanel | Canvas mounted, blank — no content drawn |
| Playing | VizPanel | Notes scroll left in real time at 60fps |
| Mode selected | VizPicker button | `var(--accent-dim)` background + `var(--accent)` outline |
| Mode idle | VizPicker button | Transparent background, muted icon |
| Scaffolded mode selected | VizPanel | Canvas mounted, blank — sketch exists but draws nothing |
| View zone loaded | Inline Monaco | 120px zone visible below `$:` line |
| After evaluate() | Inline Monaco | View zone re-added, canvas reinitializes |
