# Objective

Add `emitEvent(event: HapEvent)` to HapStream as the primary emission API, make the `hap` field optional on HapEvent (dead weight removal), and update DemoEngine and SonicPiEngine adapter to call emitEvent() directly instead of constructing fake Strudel-shaped hap objects.

# Context

The streaming path (HapStream -> useHighlighting) is separate from the queryable path (PatternScheduler -> sketches). Plans 09-01/09-02 normalized the queryable path. This plan normalizes the streaming path.

Current state:
- **HapStream.emit()** takes a raw Strudel hap + timing args, internally constructs a HapEvent by extracting fields from the hap. Works for StrudelEngine but forces DemoEngine and SonicPiAdapter to construct fake Strudel-shaped hap objects just to satisfy emit()'s signature.
- **HapEvent.hap** is `any` and NO subscriber reads it (verified in research). Dead weight.
- **sonicPiWeb** already has a clean emitEvent() pattern -- this plan brings struCode's HapStream to parity.

       Keep the eslint-disable comment for now.
       This is safe because no subscriber reads event.hap (verified in RESEARCH.md).

    2. **Add `emitEvent()` method** to HapStream class:
       ```typescript
       /**
        * Emit a pre-constructed HapEvent directly.
        * Preferred API for non-Strudel engines that don't have raw hap objects.
        */
       emitEvent(event: HapEvent): void {
         for (const handler of this.handlers) {
           try {
             handler(event)
           } catch {
             // Prevent one bad subscriber from breaking others
           }
         }
       }
       ```

    3. **Refactor emit() to use emitEvent() internally**:
       Change emit() body to construct the HapEvent then call this.emitEvent(event).
       This ensures the fan-out logic exists in one place only:
       ```typescript
       emit(hap: any, time: number, cps: number, endTime: number, audioCtxCurrentTime: number): void {
         const scheduledAheadMs = (time - audioCtxCurrentTime) * 1000
         const audioDuration = endTime - time
         const event: HapEvent = {
           hap,
           audioTime: time,
           audioDuration,
           scheduledAheadMs,
           midiNote: noteToMidi(hap?.value?.note ?? hap?.value?.n),
           s: hap?.value?.s ?? null,
           color: hap?.value?.color ?? null,
           loc: hap?.context?.locations ?? hap?.context?.loc ?? null,
         }
         this.emitEvent(event)
       }
       ```

    emit() remains for StrudelEngine (it has raw haps). emitEvent() is for engines that construct HapEvents directly.
  </action>

# Notes

- StrudelEngine continues to use emit() -- it has real Strudel haps and the extraction logic is useful. No reason to change it.
- emit() could be deprecated in a future phase once all engines migrate to emitEvent(). Not in scope here.
- The `hap?: any` field on HapEvent can be fully removed in a future cleanup phase once we're confident no external code depends on it. Making it optional is the safe first step.
- useHighlighting is completely unchanged by this plan. It consumes HapEvent fields that both emit() and emitEvent() populate identically.
