# Objective

Update all 4 queryable sketches (PianorollSketch, SpiralSketch, PitchwheelSketch, WordfallSketch) to consume NormalizedHap fields instead of raw Strudel hap internals. After this plan, no sketch reaches into `hap.whole.*`, `hap.value.*`, or `hap.endClipped`.

# Context

After 09-01, PatternScheduler.query() returns NormalizedHap[]. Sketches currently access `hap.whole.begin`, `hap.value.note`, `hap.value.color`, etc. -- these fields don't exist on NormalizedHap. TypeScript will flag every access site.

The migration is mechanical: replace deep field paths with flat NormalizedHap field names. The key complexity is in getValue()/getFreq()/getLabel() helper functions -- these do display-specific logic (fold layout, label formatting) that must be preserved but simplified.

Field mapping (old -> new):
- `hap.whole?.begin` -> `hap.begin`
- `hap.whole?.end` -> `hap.end`
- `hap.endClipped` -> `hap.endClipped`
- `hap.value?.note ?? hap.value?.n` -> `hap.note`
- `hap.value?.freq` -> `hap.freq`
- `hap.value?.s` -> `hap.s`
- `hap.value?.gain` -> `hap.gain`
- `hap.value?.velocity` -> `hap.velocity`
- `hap.value?.color` -> `hap.color`

Number() coercion is no longer needed -- normalizeStrudelHap already did it.
