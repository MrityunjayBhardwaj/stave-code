## PRE-MORTEM (the single most-likely wrong-but-green outcome)

**A binding-fixpoint / strip-walker change that passes the parity-corpus
snapshots + the editor suite but silently shifts absolute offsets so
runtime `loc` / click-to-source / highlight-on-event breaks.**

Why this is the deadliest: `parity.test.ts` strips `loc` via
`normalize.ts` (`loc-fidelity.test.ts:11-16`) — it is BLIND to offset
drift BY CONSTRUCTION. A fixpoint that splices the right subtrees but
re-bases their offsets, or a strip-walker reroute that consumes the
right tokens at a wrong absolute index, produces identical structure
(parity GREEN) while every spliced/downstream Play node's `loc.start`
drifts. The complementary gate is `loc-fidelity.test.ts` (slices every
node's `[start,end]` out of ORIGINAL source, snapshots the token text).

**THE STOP gate (mandatory, every wave that touches the parser) — stated
as a PER-FILE correlation, NOT a corpus aggregate:** run loc-fidelity
over the FULL corpus BEFORE and AFTER each parser change. The naive
aggregate form ("parity unchanged AND loc-fidelity non-empty = STOP") is
DEFEATED whenever a legitimate bare-Code→structured improvement lands on
an existing corpus file (it moves BOTH that file's parity AND its
loc-fidelity legitimately, so the corpus-aggregate parity is not
"unchanged"). The correct gate is per-file: **for every corpus file
whose parity classification is UNCHANGED, that file's loc-fidelity diff
MUST be empty. A loc-fidelity diff on a parity-unchanged file = silent
offset drift = STOP, do not commit, return to diagnosis.** A file whose
parity classification legitimately CHANGED (bare-Code → structured) is
expected to move its loc-fidelity snapshot too — that is not drift; it
is reviewed as an explained snapshot change in the PR body. The fixpoint
must NOT re-offset spliced subtrees — it reuses the
already-correctly-offset IR from the binding map (RHS parsed at its own
definition-site offset, pS:385-387).

Build hygiene (P68/PV48): editor watch is UNRELIABLE (tsup --watch dies
silently on the pre-existing `@strudel/mondo` DTS error). Per editor-src
commit: one-shot `pnpm --filter @stave/editor build`, then
`grep -c <newSymbol> packages/editor/dist/index.js` MUST be > 0 BEFORE
`git commit`.
