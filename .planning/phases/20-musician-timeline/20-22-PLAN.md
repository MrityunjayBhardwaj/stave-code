# Phase 20-22 — PLAN

## Objective

Two surgical, additive edits to `packages/editor/src/ir/parseStrudel.ts`:

1. **D-02 (the verdict lever):** widen `classifyLiteralRhs` (parseStrudel.ts:119-128)
   with a STRICT enumerated arithmetic grammar (`ARITH_RHS`, F3 verbatim) so a
   binding like `let bpm = 172/4` resolves inside `buildBindingMap`'s fixpoint.
   Today `classifyLiteralRhs('172/4')` returns `null` → `bpm` never leaves
   `pending` → the occurs-check terminal (`parseStrudel.ts:653`,
   `if (pending.size > 0) return null`) bails the WHOLE binding map → the entire
   program bareCodes **even if `bpm` is unreferenced** (RESEARCH Obs 4, RUN-proven).
   This is a whole-program-sinking bug; the fix is valuable independent of parity %.

2. **D-01 (round-trip fidelity, NOT a verdict-flip):** at recognised-numeric arms
   and opaque-wrap arg sites, substitute a literal-classified bound ident's raw
   text so `.slow(n)` with `var n = 4` emits `args="4"` not `args="n"`. The raw
   text already lives on the resolved node's `via.raw` (F1 option iii) — no new
   binding-map value shape, no PV52 ripple. This is the code-invariance the
   debugger / bidirectional-editing thesis (PV38) needs.

**This is NOT the stale "24-fallback dominant lever."** RESEARCH falsified that
premise on current main (binding-class = 8, not 24; all 4 CONTEXT-named
repros already structure). The realistic aggregate parity gain is **<1pp**, within
multi-slice noise. The phase ships two genuine correctness/fidelity wins, gated on
an arithmetic exemplar flip + a round-trip fixture + zero per-slice regression —
NOT on an aggregate % target.

## Context

### Locked decisions (SACROSANCT — plan HOW, do not relitigate)

- **D-01:** textual substitution of raw RHS text; one primitive (scalar + string);
  matcher-pure (substitute, NEVER evaluate); IR-subtree splice stays for
  Pattern-valued roots; the textual path is purely additive.
- **D-02:** widen `classifyLiteralRhs` to an enumerated arithmetic grammar
  (`/ * + -` between number tokens ONLY; NO parens, NO calls, NO idents);
  substitute raw text. STOP → pure-literals-only + follow-up issue if the grammar
  cannot be bounded cleanly.
- **D-03 (AMENDED):** multi-offset sweep (offset 0 + 100 + 250, ~150 rows, pin
  `f73b3956`). Dual gate: **crit-1 (HARD)** the arithmetic exemplar `1APcTv7DyEkW`
  flips bareCode→structured + the isolated-unreferenced case flips + the D-01
  round-trip fixture asserts `args="4"`; **crit-2 (must-not-regress ONLY)**
  blended-150 shows ZERO per-slice regression. Aggregate improvement is NOT gated.
  No bar-lowering; no chasing OUT-of-scope rows.
- **LOCKED prior (20-15/20-17):** matcher-not-interpreter — `172/4` is stored as
  raw text; Strudel evaluates it at runtime; we NEVER compute `43`.

### Design lens

**Ownership map:**
- `classifyLiteralRhs` OWNS literal recognition → produces the `{literal:true; raw}`
  arm (the source of `via.raw`). D-02 widens what it accepts.
- `buildBindingMap` OWNS the bounded fixpoint + the occurs-check terminal
  (parseStrudel.ts:653). It CONSUMES `classifyLiteralRhs`'s output post-parse. D-02
  changes nothing here — the widened recognition simply clears `pending`.
- `applyMethod` (numeric arms) / `wrapAsOpaque` OWN arg consumption. The new D-01
  primitive `substituteBoundIdentInArg(args, bindings)` READS `bindings` (PV50 — a
  trailing optional param, NEVER module state) and reads `via.raw` off the resolved
  literal-arm node. It transforms the `args` string BEFORE the arm consumes it.
- **Test-corpus ownership (auto-discovery — load-bearing for V.2):** BOTH
  `parity.test.ts` (readdirSync at :47-50, snapshot at :60-70) AND
  `loc-fidelity.test.ts` (readdirSync at :40-43) auto-discover EVERY `.strudel`
  file in `tests/parity-corpus/` at collection time and emit one `it()` per fixture.
  So the Wave 0.3 fixtures are NOT inert source — they immediately become 3 parity
  `it()` + 3 loc-fidelity `it()` blocks (+6) in the DEFAULT gate (vitest include =
  `tests/parity-corpus/**/*.test.{ts,tsx}`, confirmed at vitest.config.ts:15-19).
  The V.2 verdict spec adds its OWN count on top.

**Lifecycle (krama PK16 parser pipeline):**
1. D-02 acts at the recognition stage — inside the `buildBindingMap` fixpoint where
   `classifyLiteralRhs` runs. Widening it clears `pending`, the occurs-check
   terminal no longer bails, the map builds, the program structures.
2. D-01 acts LATER, at arg-consume time in `applyMethod` — after the binding map is
   already built and threaded. It depends on NOTHING from D-02 (a pure-literal `4`
   binding already resolves), but it is the riskier loc-fidelity surface.
3. Sequence: **D-02 first** (verdict gate + simpler, single-function widen), **D-01
   second** (riskier loc surface, fenced).

**Pre-mortem (the two HIGH risks):**
- (a) **Loc-fidelity break from textual substitution (F2, PV49).** A naive
  `args.replace(ident, raw)` re-parsed with use-site offsets computes locs against a
  string that no longer matches the user's source. Mitigation: fence the primitive
  to F2 positions (1) value operands of recognised numeric arms (loc = call-site
  range, value = parsed scalar — substituted text NEVER becomes a loc anchor) and
  (2) `via.args` round-trip text on opaque wrappers (not a loc anchor; `callSiteRange`
  is computed before `applyMethod` and is independent of `args`). FORBID position (3)
  — re-parsing a mutated string into loc-bearing leaves. String→mini-leaf cases stay
  on the EXISTING subtree-splice path (1131/1248/1430), which already handles
  `n(stringBinding)`. Enforced by a per-file loc-fidelity STOP gate (empty-diff vs
  baseline) on EVERY commit.
- (b) **D-02 scope-creep into an interpreter (P70 / #140 γ-4 pre-mortem).**
  Mitigation: the closed enumerated grammar `ARITH_RHS` (F3 verbatim) — operands are
  number tokens only, ops are exactly `/ * + -`, no recursion. STOP → pure-literal +
  follow-up issue if the corpus surfaces operand-idents (`bpm/2`) or parens.
- (c) **Inert-wall trap (MINOR-3 / V.2).** The vitest run/skip lever is the FILENAME,
  not env-gating: include matches `*.test.{ts,tsx}` ONLY; `*.spec.ts` is excluded
  (`exclude: ['node_modules', 'tests/*.spec.ts', '.next']`) AND `_`-prefixed specs
  (`_bakery-classify.spec.ts`, `_proto-d01.spec.ts`, …) run only under the separate
  `test:proto` config. A V.2 spec named `*.spec.ts` would SILENTLY never run — the
  exact inert wall the V.2 pre-mortem warns about. Mitigation: name it
  `binding-arithmetic.test.ts` and assert its `it()` count appears in the run output.

**Existence check (Chesterton):** the subtree-splice path (parseStrudel.ts:1131 /
1248 / 1430) ALREADY handles Pattern-valued + string-fn-arg cases (`n(stringBinding)`
structures today, RESEARCH Obs 3(a)). D-01 is PURELY ADDITIVE for the literal-arg-text
case — do NOT duplicate the splice; do NOT touch it.
